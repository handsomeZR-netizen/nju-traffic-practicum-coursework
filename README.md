# 南京大学交通实训课程作业

基于腾讯开悟平台智能交通信号灯调度任务完成的课程实训项目。项目围绕 `DIY` 策略槽位进行了代码重构、特征设计、训练配置迁移、云端评测联调与实验记录整理，并结合 `MCP` 协同开发工作流，形成了一套可持续迭代、可回滚、可复现实验的工程化方案。

从项目推进结果来看，这套方案已经完成了从早期 `1900+` 分数段到更高分段的稳定跃迁，并具备持续冲击 `2700+` 目标分数区间的训练与调参能力。除策略本身外，项目还重点解决了平台目录限制、云端复制粘贴协作、最小上传包生成、DIY 入口对齐等一系列工程问题。

## 项目亮点

- 基于腾讯开悟智能交通环境完成 `DIY` 智能体训练与评测闭环
- 在平台约束下将策略实现收敛到 `agent_diy/`，便于云端直接替换与回滚
- 建立了训练记录、评测复盘、配置迁移、最小上传包生成等工程流程
- 引入 `MCP` 协同开发方式，支持快速排障、结构精简、版本整理与实验追踪
- 面向高分目标持续迭代，具备冲击 `2700+` 分数区间的工程化基础

## 当前仓库内容

- `agent_diy/`: 当前主线策略实现
- `agent_ppo/`: 平台原始 PPO 参考实现
- `conf/`: 平台算法与应用配置
- `train_test.py`: 本地训练与联调入口
- `build_upload_diy_package.py`: 生成最小上传包的辅助脚本
- `upload_diy_minimal/`: 云端复制粘贴使用的最小上传包
- `实验记录与下一步计划.md`: 当前实验结论与后续计划
- `训练评测协作模板.md`: 后续协作时统一的信息模板

## 工程说明

本项目除了策略优化外，还重点完成了以下工程整理工作：

- 将原先分散的实现整理到 `DIY` 官方槽位，避免自定义目录在云端失效
- 保留平台默认结构，尽量减少对 `conf/kaiwudrl/` 与 `tools/` 的侵入式修改
- 补充初始化文档、上传说明和实验记录，降低平台环境切换成本
- 对训练环境配置进行显式管理，支持按不同评测标准快速切换

## 训练与评测说明

项目当前使用的主线方案围绕智能信号灯控制中的以下核心指标展开：

- 总得分 `score`
- 平均排队长度 `avg_queue_len`
- 平均车辆延误 `avg_delay`
- 平均车辆等待时间 `avg_waiting_time`
- 信号变化频率 `signal_change_frequency`

在不同标准下，训练环境配置会影响可比性，因此本仓库保留了实验记录文件用于标记不同环境条件下的结果，不直接混用不同标准的成绩。
