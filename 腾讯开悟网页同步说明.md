# 腾讯开悟网页同步说明

## 当前方案

仓库内已经移除 `Playwright` 与窗口坐标自动化脚本。现在只保留两类本地能力：

- 同步清单生成
- 上传批次生成

浏览器控制改由 `MCP` 完成，当前同时支持两条实验路线：

- `Browser MCP`：优先用于直接接管你当前已经登录好的 Chrome 标签页
- `Chrome DevTools MCP`：优先用于通过 Chrome DevTools Protocol 控制带调试端口的 Chrome

## 已保留的本地脚本

- `automation/sync_manifest.py`
  生成允许同步的远端清单，只保留：
  - `agent_diy/**`
  - `agent_ppo/**`
  - `conf/**`
  - `log/**`
  - `train_test.py`
- `automation/build_sync_batch.py`
  把上述文件复制到 `automation/data/sync_batch/`，并给所有 `.py` 文件注入最新的：

```python
# Uploaded at: YYYY-MM-DD HH:MM:SS CST
```

这一步不会改动本地源码，只会生成供远端上传的批次副本。

## 推荐的日常命令

如果你主要在改 `agent_diy/`，优先使用：

```bash
python automation/build_diy_sync_batch.py
```

如果这次连 `conf/**` 也改了：

```bash
python automation/build_diy_sync_batch.py --with-conf
```

如果要把所有允许范围内的变更都打进批次：

```bash
python automation/build_diy_sync_batch.py --all-allowed
```

仓库根目录还提供了一个 repo-local skill：

- `skills/kaiwu-diy-upload/`

这个 skill 已经把本项目的远端路径限制、DIY 优先策略、`chrome-devtools-mcp` 工作流和推荐命令写死，后续在这个仓库里继续协作时可以直接复用。

## 已移除的旧链路

以下基于浏览器自动化的脚本已经删除：

- `automation/tencentarena_login.py`
- `automation/tencentarena_sync.py`
- `automation/tencentarena_sync_ui.py`
- `automation/tencentarena_selectors.py`

因此当前仓库不再支持：

## 使用方式

### 1. 生成清单

```bash
python automation/sync_manifest.py --json-out automation/data/sync_manifest.json
```

### 2. 生成上传批次

```bash
python automation/build_sync_batch.py
```

默认输出：

- 批次目录：`automation/data/sync_batch/`
- 批次清单：`automation/data/sync_batch_manifest.json`

### 3. 用 MCP 接管浏览器并执行上传

#### 路线 A：Browser MCP

适用场景：

- 想直接操作“你当前已经登录好的 Chrome”
- 想避免再次登录
- 更看重当前标签页接管体验

前置条件：

1. 在 Chrome 中安装 `Browser MCP` 扩展
2. Codex 已加载 `browsermcp` server
3. 当前标签页已经停在腾讯开悟 `代码开发` 页面

#### 路线 B：Chrome DevTools MCP

适用场景：

- 想通过 DevTools 协议获得更细的 DOM / 调试能力
- 愿意使用带调试端口启动的 Chrome

推荐启动命令示例：

```powershell
& "C:\Program Files\Google\Chrome\Application\chrome.exe" `
  --remote-debugging-port=9222 `
  --user-data-dir="$env:LOCALAPPDATA\ChromeDevtoolsMcpProfile"
```

然后在这个 Chrome 实例里登录腾讯开悟，再让 Codex 通过 `chrome_devtools` MCP 接管页面。

## Codex MCP 配置

本机 `~/.codex/config.toml` 已增加：

- `mcp_servers.browsermcp`
- `mcp_servers.chrome_devtools`

如果当前 Codex 会话还看不到这两个 MCP，需要重启 Codex 让新配置生效。

## 对比建议

建议按同一任务比较两条路线：

1. 打开腾讯开悟 `代码开发` 页面
2. 打开远端 `train_test.py`
3. 检查前几行是否能稳定读取
4. 试一次小范围编辑与保存

一般判断顺序建议是：

1. 是否能直接接管当前已登录页面
2. 是否减少重复登录
3. 白屏或扩展加载时是否稳定
4. 打开 VS Code Web 文件是否顺手

## 注意事项

- 远端只接受 `agent_diy/`、`agent_ppo/`、`conf/`、`log/`、`train_test.py`
- `.py` 时间戳只写入上传批次，不改本地源码
- 如果腾讯页面自动清除不合规文件，本地脚本不额外实现删除逻辑
