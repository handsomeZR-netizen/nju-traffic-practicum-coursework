# 训练评测协作模板

这份模板用于后续和我协作时统一信息格式。当前默认主线已经切到 `normal`，并且进入“**低评测消耗**”阶段，所以以后你每次发结果时，第一件事就是先标清楚：

- 当前环境层级
- 当前版本名
- 当前 checkpoint
- 剩余评测次数

原则很简单：

- 训练时给训练信息
- 评测时给评测信息
- 报错时给完整报错和目录状态
- 每次尽量带配置全文，不要只口述
- 同一环境内再比较分数，不同环境只做历史参考
- 评测次数有限时，优先确认最强候选，不做大范围横向筛点

---

## 0. 每次先发这四个标签

以后你不管是训练、评测还是报错，最前面先补这四项：

```text
环境层级：easy / normal
当前版本名：例如 normal-diy-30-30
checkpoint：例如 27min / 60min / latest
剩余评测次数：
```

如果是接着某个旧模型继续训，再补一项：

```text
checkpoint 来源：例如从 normal-diy-30-30 继续训练 60 min
```

## 7. 推荐协作顺序

以后我们尽量按这个顺序配合：

1. 你先发环境层级、版本名、checkpoint、剩余评测次数
2. 你再发环境配置全文
3. 你补本次改动范围或 checkpoint 来源
4. 你发训练或评测截图
5. 你补均分、每局分数、关键指标或完整报错
6. 我再给你下一轮修改建议或直接改代码
